# 变更提案：页面有一个比较严重的bug，4月份有消费数据，但是统计显示没有任何消费数据

## 变更原因

Bug 反馈：用户在 4 月份有多笔消费记录，但在统计页面查看本月（4月）数据时，显示"该时间段暂无记录"。

## 变更类型

- [x] 代码级（Bug 修复）

## 变更范围

- 影响的文件：src/main/database.js (getStatistics 函数)
- 影响的场景：S02-查看统计报表

## 变更概述

**问题根因**：在 `getStatistics` 函数中，按月份查询时使用了错误的比较逻辑：

```javascript
dateFilter = "AND strftime('%Y-%m', r.created_at) = ?";
dateParams.push(month + '-01');
```

这里 `strftime('%Y-%m', r.created_at)` 返回的是 "2026-04" 这样的月份字符串，而传入的参数是 "2026-04-01"，SQL 比较的是 "2026-04" = "2026-04-01"，结果永远为 false。

**修复方案**：修改传入参数为 `month`（如 "2026-04"），使比较逻辑正确匹配。
