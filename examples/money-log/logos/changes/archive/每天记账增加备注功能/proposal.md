# 变更提案：每天记账增加备注功能

## 变更原因

用户在日常记账时，常常需要对某笔支出添加额外说明（如"晚餐火锅AA制"、"打车去医院"）。目前的记账功能只支持金额和分类，缺少备注输入功能，导致用户需要额外记录或记忆消费细节。

## 变更类型

- [x] 代码级

## 变更范围

- 影响的需求文档：01-requirements.md (S01 场景)
- 影响的功能规格：01-main-design.md (S01 记账页)
- 影响的业务场景：S01-快速记账.md
- 影响的 API：local-api.yaml (records:save 已支持 remark 字段)
- 影响的 DB 表：records.remark (字段已存在)
- 影响的测试用例：S01-test-cases.md (UT-S01-06 已覆盖)

## 变更概述

在记账页面（S01 快速记账）的金额输入区域下方，添加一个可选的备注输入框，用户可以输入最多 200 字符的备注信息。记账时将备注一并保存到数据库。

**现状**：数据库和 API 已支持 remark 字段，但前端 UI 缺少输入入口。

**目标**：添加备注输入框，实现完整记账流程。
