# 变更提案：我希望能够看到每月的消费明细

## 变更原因

用户反馈：目前的统计页面只显示每日的支出趋势和分类汇总，但无法看到具体每一笔消费的明细（如"4月5日 餐饮 ¥35 午餐"）。用户希望能够点击查看具体的消费记录列表，满足追溯消费明细的需求。

- [x] 代码级

- 影响的 DB 表：records 表 (已存在)

## 变更概述

在统计页面（S02 查看统计报表）的分类汇总列表下方，添加「消费明细」区域，显示当前时间范围内的所有记账记录列表，每条显示：日期、分类、金额、备注。

**现状**：统计页只显示聚合数据（柱状图 + 分类汇总），无明细列表。

**目标**：在分类汇总下方增加明细列表，支持折叠/展开。
