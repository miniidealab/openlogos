## 三、场景交互规格

### S01: 快速记账

**涉及页面**：记账页

**页面布局**：
```
┌─────────────────────────────────────────────────────┐
│                   快速记账                           │
├─────────────────────────────────────────────────────┤
│                                                     │
│   分类选择区（横向滚动）                             │
│   ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│   │餐饮│ │交通│ │购物│ │娱乐│ │居住│ │医疗│ ...   │
│   └────┘ └────┘ └────┘ └────┘ └────┘ └────┘      │
│                                                     │
│   金额输入区                                         │
│   ┌─────────────────────────────────────────────┐   │
│   │                    ¥ 0.00                   │   │
│   └─────────────────────────────────────────────┘   │
│                                                     │
│   ┌─────────────────────────────────────────────┐   │
│   │                    记                       │   │
│   └─────────────────────────────────────────────┘   │
│                                                     │
│   今日概览（可选展示）                               │
│   今日支出：¥ 120.00                               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**交互细节**：

| 元素 | 规格 |
|------|------|
| 分类标签 | 横向排列，支持滚动，选中态为蓝色背景+白色文字 |
| 金额输入框 | 数字键盘输入，支持小数点，默认 ¥0.00，点击清空 |
| 「记」按钮 | 满宽主按钮，根据条件禁用/启用 |
| 成功后反馈 | 金额区域短暂显示「已记录 ✓」动画（1s），金额清零 |

**状态规则**：

| 状态 | 「记」按钮 |
|------|-----------|
| 未选分类 + 金额为0 | 禁用 |
| 已选分类 + 金额为0 | 禁用 |
| 已选分类 + 金额>0 | 启用 |
| 未选分类 + 金额>0 | 禁用 |

**交互级验收条件**：

##### 正常：完整记账
- **GIVEN** 用户在记账页，无分类选中，金额显示 ¥0.00
- **WHEN** 用户点击「餐饮」分类标签，输入「35.5」，点击「记」
- **THEN** 金额区显示「已记录 ✓」1秒，金额恢复 ¥0.00，分类保持「餐饮」选中

##### 正常：连续记账（同分类）
- **GIVEN** 用户刚完成一笔「餐饮」¥35.5
- **WHEN** 用户继续输入「12」点击「记」
- **THEN** 显示「已记录 ✓」，记录第二条

##### 异常：未选分类
- **GIVEN** 用户在记账页，无分类选中
- **WHEN** 用户输入「100」点击「记」
- **THEN** 按钮保持禁用状态，或点击后 Toast 提示「请先选择分类」

##### 异常：金额格式
- **GIVEN** 用户已选「餐饮」
- **WHEN** 用户输入「abc」或「-50」
- **THEN** 输入框拒绝显示无效字符

- **GIVEN** 用户有 4 月餐饮记录
- **WHEN** 用户选择分类「餐饮」
- **THEN** 柱状图显示 4 月每日餐饮支出，列表仅显示「餐饮」一项

##### 异常：时间范围无数据
- **GIVEN** 用户选择「1 月」但无记录
- **WHEN** 用户选择 1 月
- **THEN** 显示空状态插图 + 「该时间段暂无记录，开始记账吧」

```
├─────────────────────────────────────────┤

│   │                                 │   │

│                                         │
│   ┌─────────────────────────────────┐   │

│                                         │
└─────────────────────────────────────────┘
```

**交互细节**：

| 元素 | 规格 |
|------|------|
| 预设分类 | 不可删除，仅可编辑名称 |
| 自定义分类 | 可编辑/删除 |
| 删除按钮 | 仅自定义分类显示，点击弹出确认 |
| 新增按钮 | 满宽次要按钮 |

**交互级验收条件**：

##### 正常：新增分类
- **GIVEN** 用户在分类管理页
- **WHEN** 点击「+ 新增分类」，输入「咖啡」，点击「保存」
- **THEN** 弹窗关闭，分类列表新增「咖啡」，记账页可选

##### 正常：编辑分类
- **GIVEN** 用户自定义了「咖啡」分类
- **WHEN** 点击「咖啡」的编辑，输入「饮品」，点击「保存」
- **THEN** 分类名更新为「饮品」，记账页同步

##### 正常：删除分类
- **GIVEN** 用户自定义了「咖啡」分类（3条记录）
- **WHEN** 点击「咖啡」的删除，确认删除
- **THEN** 「咖啡」移除，3条记录归入「其他」

##### 异常：重名
- **GIVEN** 已存在「餐饮」分类
- **WHEN** 新增分类输入「餐饮」，点击「保存」
- **THEN** 弹窗内显示「分类名已存在」，不保存

##### 异常：删除预设分类
- **GIVEN** 「餐饮」是预设分类
- **WHEN** 点击「餐饮」的删除按钮
- **THEN** 删除按钮不存在或不可点击

|------|------|

| 密码错误反馈 | 抖动动画 + 红色提示「密码错误」 |
| 忘记密码 | 底部链接，点击弹出确认对话框 |
| 密码开关 | 开启密码锁后显示「修改密码」入口 |

**交互级验收条件**：

##### 正常：首次开启密码锁
- **GIVEN** 用户在密码锁设置页，密码锁关闭
- **WHEN** 开启开关，输入密码「123456」，确认密码「123456」，点击「保存」
- **THEN** 关闭弹窗，开关变为开启状态

##### 正常：正确解锁
- **GIVEN** 已开启密码锁，密码为「123456」
- **WHEN** 用户打开 App，输入「123456」
- **THEN** 进入主页面

##### 正常：关闭密码锁
- **GIVEN** 已开启密码锁
- **WHEN** 在设置页关闭开关，输入原密码「123456」
- **THEN** 开关关闭，下次打开无需密码

##### 异常：密码错误
- **GIVEN** 已开启密码锁，正确密码「123456」
- **WHEN** 用户输入「000000」
- **THEN** 输入框抖动，显示「密码错误」

##### 异常：忘记密码
- **GIVEN** 已开启密码锁，用户忘记密码
- **WHEN** 在解锁页点击「忘记密码」，确认清除数据
- **THEN** 清除所有本地数据，重置 App

##### 异常：密码不一致
- **GIVEN** 在设置密码弹窗
- **WHEN** 输入密码「123456」，确认密码「654321」
- **THEN** 显示「两次密码不一致」
