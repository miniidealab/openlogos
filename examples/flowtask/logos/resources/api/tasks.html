Commands

create_task

创建新任务

来源：S02.2 Step 5 → Step 8。将新任务写入 tasks 表，done 默认为 false。

Params

名称类型必填说明
user_id integer ● 当前登录用户 ID，来自 Zustand Store
name string maxLen: 100, minLen: 1 ● 任务名称（必填，前端已校验非空）
category_id integer 分类 ID，null 表示无分类
note string maxLen: 500 备注（可选）

Returns → TaskItem

任务完整记录

名称类型必填说明
id integer ● 任务主键 ID
user_id integer ● 所属用户 ID
category_id integer 所属分类 ID，null 表示无分类
name string maxLen: 100 ● 任务名称
note string maxLen: 500 备注
done boolean ● 完成状态，false = 未完成，true = 已完成
created_at string ● 创建时间，ISO 8601 字符串
updated_at string ● 最后更新时间，ISO 8601 字符串

Errors

DB_WRITE_ERROR EX-6.1
保存失败，请重试
来源：S02.2 Step 6 INSERT 操作失败

update_task

更新任务内容

来源：S02.3b Step 5 → Step 8。更新任务的名称、分类、备注字段，同步更新 updated_at。

Params

名称类型必填说明
task_id integer ● 目标任务 ID
name string maxLen: 100, minLen: 1 ● 更新后的任务名称
category_id integer 更新后的分类 ID，null 表示无分类
note string maxLen: 500 更新后的备注

Returns → TaskItem

任务完整记录

名称类型必填说明
id integer ● 任务主键 ID
user_id integer ● 所属用户 ID
category_id integer 所属分类 ID，null 表示无分类
name string maxLen: 100 ● 任务名称
note string maxLen: 500 备注
done boolean ● 完成状态，false = 未完成，true = 已完成
created_at string ● 创建时间，ISO 8601 字符串
updated_at string ● 最后更新时间，ISO 8601 字符串

Errors

TASK_NOT_FOUND EX-6.1（编辑场景）
任务不存在
UPDATE 影响行数为 0，任务已被删除（并发场景）

update_task_status

切换任务完成状态

来源：S02.3a Step 2 → Step 5。专用于切换 done 字段，与更新内容分离，保持职责单一。

Params

名称类型必填说明
task_id integer ● 目标任务 ID
done boolean ● 目标状态：true = 标记完成，false = 取消完成

Returns

名称类型必填说明
task_id integer ●
done boolean ●

Errors

TASK_NOT_FOUND EX（并发）
任务不存在
UPDATE 影响行数为 0

delete_task

删除任务

来源：S02.3c Step 4 → Step 7。物理删除，不可恢复。WHERE 条件同时包含 user_id 防止越权。

Params

名称类型必填说明
task_id integer ● 目标任务 ID

Returns

名称类型必填说明
success boolean ●

Errors

TASK_NOT_FOUND EX-6.1
任务不存在（静默处理）
来源：S02.3c Step 6 DELETE 影响行数为 0，前端静默处理，结果最终一致
