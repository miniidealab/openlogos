# FlowTask 功能规格文档

> 最后更新：2026/04/07
> 产品类型：桌面 GUI 应用（macOS / Windows）
> 原型格式：HTML 交互页面

---

## 一、信息架构

### 页面结构

```
FlowTask
├── 认证模块
│   ├── 注册页（首次启动）
│   └── 登录页（已有账号）
└── 主应用模块（登录后）
    ├── 任务主界面（默认页）
    │   ├── 分类侧边栏
    │   ├── 任务列表区
    │   └── 新建/编辑任务面板
    └── 账号设置页
        └── 修改密码表单
```

### 路由逻辑

| 条件 | 跳转目标 |
|------|----------|
| 本地无账号数据 | 注册页 |
| 本地有账号，未登录 | 登录页 |
| 已登录 | 任务主界面 |

## 二、场景交互规格

### S01：新用户注册并开始使用

**涉及页面**：注册页 → 任务主界面

**交互流程**：
1. APP 首次启动，检测本地无账号数据 → 显示注册页
2. 注册页包含：用户名输入框、密码输入框、确认密码输入框、注册按钮、"已有账号？去登录"链接
3. 实时校验：用户名 2-20 字符、密码 ≥8 位、两次密码一致
4. 提交成功 → 本地创建账号（密码加密存储）→ 自动登录 → 跳转任务主界面
5. 任务主界面首次进入显示空状态引导："还没有任务，点击 + 创建第一个任务"

**字段规格**：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 用户名 | 文本输入 | 是 | 2-20 个字符，不含特殊符号 |
| 密码 | 密码输入 | 是 | ≥8 位字符 |
| 确认密码 | 密码输入 | 是 | 必须与密码一致 |

**验收标准（交互级）**：

##### 正常：注册成功
- **GIVEN** 注册页所有字段为空
- **WHEN** 用户输入用户名"xiaoli"、密码"Pass1234"、确认密码"Pass1234"，点击"注册"
- **THEN** 注册按钮进入 loading 状态，500ms 内跳转任务主界面，显示空状态引导文案

##### 异常：用户名已存在
- **GIVEN** 本地已有用户名"xiaoli"
- **WHEN** 用户输入用户名"xiaoli"，点击"注册"
- **THEN** 用户名输入框下方显示红色提示"该用户名已被使用，请换一个"，焦点回到用户名输入框

##### 异常：密码少于 8 位
- **GIVEN** 用户在注册页
- **WHEN** 用户输入密码"abc123"（6位），点击"注册"
- **THEN** 密码输入框下方显示"密码至少需要 8 位字符"，不提交

##### 异常：两次密码不一致
- **GIVEN** 用户在注册页
- **WHEN** 用户输入密码"Pass1234"、确认密码"Pass5678"，点击"注册"
- **THEN** 确认密码输入框下方显示"两次输入的密码不一致"，不提交

### S02：用户登录并管理当日任务

**涉及页面**：登录页 → 任务主界面（含新建/编辑任务面板）

#### S02.1 用户登录

**交互流程**：
1. APP 启动，检测本地有账号 → 显示登录页
2. 登录页包含：用户名输入框、密码输入框、登录按钮
3. 提交成功 → 跳转任务主界面，显示该用户的未完成任务列表

**验收标准（交互级）**：

##### 正常：登录成功
- **GIVEN** 用户在登录页，字段为空
- **WHEN** 用户输入正确的用户名和密码，点击"登录"
- **THEN** 登录按钮进入 loading 状态，跳转任务主界面，任务列表加载完成

##### 异常：密码错误
- **GIVEN** 用户在登录页
- **WHEN** 用户输入正确用户名、错误密码，点击"登录"
- **THEN** 密码输入框下方显示"用户名或密码错误"，密码框清空，焦点回到密码框

#### S02.2 创建任务

**交互流程**：
1. 用户点击右下角"+"按钮 → 右侧滑出新建任务面板
2. 面板包含：任务名称输入框（必填）、分类下拉选择（可选）、备注文本域（可选）、保存按钮、取消按钮
3. 点击"保存" → 任务出现在列表顶部，面板关闭

**字段规格**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务名称 | 文本输入 | 是 | 最多 100 字符 |
| 分类 | 下拉选择 | 否 | 来自用户已创建的分类，默认"无分类" |
| 备注 | 多行文本 | 否 | 最多 500 字符 |

**验收标准（交互级）**：

##### 正常：创建任务成功
- **GIVEN** 用户在任务主界面，新建任务面板已打开
- **WHEN** 用户输入任务名称"完成项目报告"，选择分类"工作"，点击"保存"
- **THEN** 面板关闭，任务列表顶部出现新任务卡片，显示任务名称和"工作"分类标签

##### 异常：任务名称为空
- **GIVEN** 新建任务面板已打开
- **WHEN** 用户未填写任务名称，点击"保存"
- **THEN** 任务名称输入框下方显示"任务名称不能为空"，面板不关闭

#### S02.3 完成/编辑/删除任务

**交互流程**：
- **完成**：点击任务卡片左侧勾选框 → 任务显示删除线样式，移至列表底部"已完成"区域
- **编辑**：点击任务卡片 → 右侧滑出编辑面板（与新建面板相同布局，预填当前数据）→ 修改后点击"保存"
- **删除**：鼠标悬停任务卡片 → 显示删除图标 → 点击 → 弹出确认对话框"确认删除该任务？此操作不可恢复" → 确认删除

### S03：用户按分类整理任务

**涉及页面**：任务主界面（分类侧边栏 + 分类管理弹窗）

**交互流程**：
1. 左侧侧边栏显示分类列表，顶部固定"全部"选项
2. 侧边栏底部有"管理分类"入口 → 打开分类管理弹窗
3. 分类管理弹窗：显示已有分类列表、新增分类输入框、每个分类右侧有删除按钮
4. 点击侧边栏某分类 → 任务列表筛选显示该分类任务

**字段规格**：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 分类名称 | 文本输入 | 是 | 1-20 字符，同一用户下唯一 |

**验收标准（交互级）**：

##### 正常：创建分类
- **GIVEN** 分类管理弹窗已打开
- **WHEN** 用户输入"工作"，点击"添加"
- **THEN** "工作"出现在分类列表中，同时出现在侧边栏和任务新建/编辑的分类下拉中

##### 正常：按分类筛选
- **GIVEN** 用户在任务主界面，侧边栏当前选中"全部"
- **WHEN** 用户点击侧边栏"工作"分类
- **THEN** 任务列表只显示分类为"工作"的任务，侧边栏"工作"高亮，顶部显示"工作 (3)"

##### 异常：删除含任务的分类
- **GIVEN** 分类管理弹窗已打开，"工作"分类下有 3 个任务
- **WHEN** 用户点击"工作"右侧删除按钮
- **THEN** 弹出确认提示"该分类下有 3 个任务，删除后这些任务将变为未分类，是否继续？"，用户点击"确认"后分类删除，任务保留

### S04：用户修改账号密码

**涉及页面**：账号设置页（从主界面右上角头像/设置入口进入）

**交互流程**：
1. 用户点击主界面右上角"设置"图标 → 跳转账号设置页
2. 账号设置页显示：当前用户名（只读）、修改密码区域
3. 修改密码区域包含：当前密码输入框、新密码输入框、确认新密码输入框、保存按钮
4. 提交成功 → 显示"密码修改成功"提示，停留在当前页

**字段规格**：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 当前密码 | 密码输入 | 是 | 与本地存储密码匹配 |
| 新密码 | 密码输入 | 是 | ≥8 位，不能与当前密码相同 |
| 确认新密码 | 密码输入 | 是 | 必须与新密码一致 |

**验收标准（交互级）**：

##### 正常：修改密码成功
- **GIVEN** 用户在账号设置页
- **WHEN** 用户输入正确的当前密码、新密码"NewPass99"、确认新密码"NewPass99"，点击"保存"
- **THEN** 三个输入框清空，页面顶部显示绿色提示条"密码修改成功"，3 秒后自动消失

##### 异常：当前密码错误
- **GIVEN** 用户在账号设置页
- **WHEN** 用户输入错误的当前密码，点击"保存"
- **THEN** 当前密码输入框下方显示"当前密码不正确"，当前密码框清空

##### 异常：新密码与当前密码相同
- **GIVEN** 用户在账号设置页，已正确输入当前密码
- **WHEN** 用户输入的新密码与当前密码相同，点击"保存"
- **THEN** 新密码输入框下方显示"新密码不能与当前密码相同"

## 三、通用交互规范

### 表单校验
- 错误提示显示在对应输入框正下方，红色文字，字号 12px
- 点击输入框时错误提示消失，失焦或提交时重新校验
- 必填字段为空时，输入框边框变红

### 加载状态
- 所有提交操作按钮进入 loading 状态（禁用 + 显示加载动画），防止重复提交

### 空状态
- 任务列表为空时显示插图 + 引导文案"还没有任务，点击 + 开始添加"
- 分类筛选结果为空时显示"该分类下暂无任务"

### 确认对话框
- 删除操作统一使用确认对话框，包含操作描述和"确认"/"取消"两个按钮
- 确认按钮为红色（危险操作），取消按钮为默认样式
