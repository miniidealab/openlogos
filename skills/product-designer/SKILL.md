# Skill: Product Designer

> [WIP] 辅助撰写产品设计文档，定义功能规格（含 GIVEN/WHEN/THEN 验收条件），生成 HTML 产品原型。

## 触发条件

- 用户要求写产品设计文档或功能规格
- 用户要求画原型或生成 HTML 原型
- 用户提到 "Phase 2"、"产品设计层"、"WHAT"
- 已有需求文档，需要开始设计解决方案

## 核心能力

1. 设计信息架构（页面结构、导航层级）
2. 撰写功能规格（含 GIVEN/WHEN/THEN 功能验收条件）
3. 生成 HTML 产品原型（使用 AI 直接生成可交互页面）
4. 建立需求追溯矩阵（每个功能追溯到需求条目）
5. 制定全局设计规范

## 执行步骤

> 详细步骤待完善。核心流程：
>
> 1. 分析 Phase 1 需求文档，提取需要设计的功能列表
> 2. 设计信息架构
> 3. 为每个功能编写规格 + GIVEN/WHEN/THEN 验收条件
> 4. 生成 HTML 原型
> 5. 建立需求追溯矩阵
> 6. 输出完整的产品设计文档

- 设计文档与原型成对出现：`{序号}-{名称}-design.md` + `{序号}-{名称}-prototype.html`
